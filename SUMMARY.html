LLM Speedrun Submission Summary

Architecture & Parameter Overview

The final model is an autoregressive Transformer (GPT-style) configured to hit ~1.87M parameters, safely under the 2,000,000 cap. The core architecture relies on 4 transformer blocks, a hidden dimension of 192, and 6 attention heads. Weight tying was utilized between the token embedding and the output head to save parameter space, allowing for wider dense layers within the blocks.

Optimal Hyperparameters

  • Optimizer: AdamW (Weight decay: 0.1)
  • Learning Rate: Max 2e-3 with Cosine Annealing and 150-step Warmup
  • Batch Size: 32
  • Context Window: 256 tokens
  • Dropout: 0.0 (Removed to maximize short-term adaptation)

Methodology & Justification

The primary metric of Bits Per Byte (bpb) was best optimized by treating this as an underfitting problem rather than an overfitting problem. A short 2,000-step run requires highly aggressive learning dynamics. By disabling dropout, increasing the batch size to 32, and raising the peak learning rate to 2e-3, the model was forced to converge rapidly. Experimental runs showed that extending the context window to 512 tokens degraded the bpb, proving that the model did not have enough steps to learn longer positional embeddings efficiently.

Human vs. Machine Contribution

Machine Assistance: The AI coding assistant was responsible for generating the boilerplate PyTorch transformer code, implementing the cosine learning rate scheduler math, modifying the tokenizer to ensure strict UTF-8 compliance without bloating the vocabulary, and generating this HTML summary document.

Human Intervention: Human effort was focused on scientific testing, tracking the RUNLOG, navigating the local directory file-path issues to ensure the pipeline executed correctly, and making the strategic architectural decisions (such as realizing dropout was hindering short-term performance and reverting the block_size expansion when it failed to yield dev set improvements).
